# SystemAnalysis Examples

This file contains examples of system analysis documents.

## Example 1: Payment System Analysis

### Project Background

**业务背景：**
开发一个预付卡系统，支持用户充值、消费、退款等核心功能。目标用户为平台商家和消费者。

**开发背景：**
全新产品开发，采用微服务架构，基于现有支付平台构建。

### System Goals

- 支持每日100万笔交易
- 交易处理延迟 < 200ms
- 系统可用性 > 99.99%

### Functional Requirements

| 优先级 | 功能列表 | 备注 |
|--------|----------|------|
| P1 | 用户充值 | 支持银行卡、余额充值 |
| P1 | 消费支付 | 支持扫码、被扫、SDK |
| P1 | 退款 | 支持全额、部分退款 |
| P2 | 充值记录查询 | 支持时间范围查询 |
| P2 | 交易对账 | 日终对账功能 |

### Database Design

**核心表结构：**

| 表名 | 说明 |
|------|------|
| prepay_card | 预付卡基本信息 |
| prepay_card_balance | 预付卡余额 |
| prepay_card_transaction | 交易流水 |
| prepay_card_refund | 退款记录 |

### Risk Analysis

**资损风险点：**

| 风险点描述 | 防控措施 |
|------------|----------|
| 并发退款超额 | 使用分布式锁+余额校验 |
| 充值金额错误 | 双向校验+回调确认 |

**安全风险点：**

| 风险点描述 | 防控措施 |
|------------|----------|
| 银行卡信息泄露 | 敏感数据加密存储 |
| API被滥用 | 限流+签名验证 |

## Example 2: Order System Analysis (Partial)

### Business Analysis

**业务用例：创建订单**

1. 用户选择商品加入购物车
2. 提交订单，选择支付方式
3. 系统创建订单，记录订单信息
4. 用户完成支付
5. 系统更新订单状态为已支付

**测试用例：**

| 序号 | 场景描述 | 关注点 |
|------|----------|--------|
| 1 | 正常创建订单 | 订单号生成、状态正确 |
| 2 | 库存不足 | 提示库存不足、订单创建失败 |
| 3 | 重复提交 | 防重校验、避免重复订单 |

### Stability Analysis

| 分析内容 | 分析结果 | 处理意见 |
|----------|----------|----------|
| 数据库单点 | 订单表数据量大 | 建议分库分表 |
| 缓存依赖 | 库存使用Redis | 需保证缓存一致性 |
| 消息队列 | 订单状态同步 | 需保证消息不丢失 |
