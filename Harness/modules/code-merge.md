## Fast Feedback Economics
**Traditional World:** Fixing expensive, waiting cheap  
**Agent World:** Fixing cheap, waiting expensive → Fast release, fast exposure, fast fix

## Three Key Actions

1. **Minimize Blocking Gates** - Keep only minimal blocking gates
2. **Keep PR Lifecycle Short** - High-frequency small-step fast merges, collision probability extremely low
3. **Never Let Flaky Tests Block Progress Indefinitely** - Use re-run mechanism instead of blocking

## Visual Verification Strategy (How to verify "UI looks correct")

Agent cannot "see" like a human, so we must rely on **proxies for vision**:

1. **DOM Snapshots (Lightweight)**
   - **Tool**: Jest Snapshots (`expect(tree).toMatchSnapshot()`)
   - **Check**: Did the HTML structure change unexpectedly?
   - **Action**: If changed, diff the HTML. If intentional, update snapshot.

2. **Visual Regression (Heavyweight)**
   - **Tool**: Percy, Chromatic, or Playwright screenshots
   - **Check**: Pixel-by-pixel comparison against baseline.
   - **Action**: Check the diff image generated by CI.

3. **Preview URL (Human Check)**
   - **Tool**: Vercel Preview / Netlify Preview
   - **Action**: Output the Preview URL in the final report for human verification.

**Applicability:** Only holds when agent code output far exceeds human attention.
